Our Expertise

Containerization

The concept of the cloud is solidly anchored in todays world of IT. No more bare metal machines, just software control panes where turnkey platform architectures are realized and controlled. While the deployment of and operations on applications can still be done using operating system level operations the focus has slowly shifted to containerized applications. In short an application is packaged in an image, a stripped virtual machine of sorts, and distributed to the relevant control pane; be it AWS, Digital Ocean, Azure or google cloud engine. These images are inherently cross platform, meaning that when the container technology used to build the images is available, it will run. These images can be used to instantiate a container that, during it's lifespan will host the application as a single process.

Microservices

One of the most trending subject of todays world of integration. Microservices are a form of a Service Oriented Architecture where "micro" does not necissarily mean "small" and "service" does not necissarily has the meaning as we use it so often while discussing systems integration. Microservices above all dictates an architectural principle of designing applications "Domain Driven". Essentially a microservice can contain more than one service, it can even contain an entire application and it is not per se re-usable, it is however a singular block of functionality.
